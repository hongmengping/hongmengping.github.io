JS的缺点

  • 允许重复的变量声明：导致数据被覆盖

    var a = 1;
    function print() {
    console.log(a)
    }
    //假设有n行代码
    var a = 2;
    print()

    a = 2 会覆盖 a = 1 由于js这种事弱语言，变量重复不会报错，导致代码量一旦增加，就容易出现问题。而且这种问题很难发现。

  • 变量提升：怪异的数据访问、闭包问题
    • 1.变量提升：怪异的数据访问
      if(Math.random() < 0.5>) {
      var a = “abc”;
      console.log(a);
      }else {
      console.log(a);
      }
      console.log(a);
      代码的字面意思：
      如果随机数小于0.5，则声明变量a，赋值为abc，输出a
      如果随机数不小于0.5，则输出a

    但是js在开始运行之前会进行变量提升，导致这段代码的意思变为
    先声明一个变量a
    如果随机数小于0.5，赋值为abc，输出a
    否则,输出a
    由此可见 js的变量提升会让数据访问变怪异

    • 2.变量提升，闭包
      点击每一个按钮都只会输出11 原因是var i提升到全局，又因为js有变量提升以及预编译，导致循环的每一个i都是全局的这个i，预编译让i提前编译成11，所以导致不管点击哪个按钮都只会输出11。
      var div = document.getElementById(“divButtons”);
      for(var i = 1; i <= 10; i ++) {
      var btn = document.createElement(“button”);
      btn.innerHTML = “按钮” + i;
      div.appendChild(btn);
      btn.onclick=function() {
      console.log(i);
      }
      }

    js中解决闭包 使用立即执行函数
    var div = document.getElementById(“divButtons”);
    for(var i = 1; i <= 10; i ++) {
    var btn = document.createElement(“button”);
    btn.innerHTML = “按钮” + i;
    div.appendChild(btn);
    (function(i) {
    btn.onclick=function() {
    console.log(i);
    }
    })
    }

  • 全局变量挂载到全局对象：全局对象成员污染问题

    如果在全局定义变量，最后会归属于window。如果该变量和window中的变量相同，则会覆盖window中的变量，造成严重的错误。
